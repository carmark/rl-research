奖励建模对比 (参考 arXiv 2509.02547)

RL 算法族谱

PPO 族 (需要 Critic)

  • PPO-Clip: verl, ROLL, SLIME, Seer
  • PPO-KL: verl
  • Turn-PPO (Turn级): 前沿研究

GRPO 族 (无 Critic)

  • GRPO: verl, ROLL, SLIME, MILES, Seer
  • DAPO (解耦Clip): verl, ROLL, SLIME
  • GRPO+TIS/MIS: MILES
  • GSPO: verl, SLIME

自研变体

  • CISPO (裁剪IS权重): Forge
  • SAPO (Swarm采样): rl-swarm

DPO 族 (无 RL 循环)

  • DPO/SimPO/KTO: verl 辅助
  • 局限: 不适合多步推理和工具使用

趋势: GRPO 通过消除 Critic 模型减少约 50% 资源开销。DAPO 进一步解决长 CoT 场景的熵坍塌。 奖励建模从人类标注 → RLVR (可验证奖励) 演进。模块化堆栈: SFT → DPO 对齐 → GRPO/DAPO 推理。

verl - HybridFlow

  • 单控制器+SPMD 混合编程模型
  • 3D-HybridEngine 权重重分片
  • TransferQueue 控制流/数据流解耦
  • 最大社区和最广泛的后端支持

ROLL - ROLLMUX/ROLLART

  • ROLLMUX 阶段级 GPU 复用 (1.84x)
  • ROLLART 无状态组件 Serverless 化
  • AutoDeviceMapping 灵活部署
  • 共置/解耦/Serverless 三模式

SLIME - APRIL

  • 主动部分 Rollout 长尾管理
  • SGLang 原生集成
  • SlimeRouter 负载均衡
  • 唯一验证 355B MoE RL 训练

MILES - R3/FP8

  • R3 路由回放解决 MoE 不一致
  • 首个端到端 FP8 框架
  • 在线 SFT 投机解码 (25%+)
  • INT4 QAT: 1TB 单机推理

Forge - CISPO/Prefix Tree

  • CISPO: 全 Token 梯度 (2x DAPO)
  • Prefix Tree Merging (~40x 加速)
  • 三层中间件完全解耦
  • 数百种 Agent 脚手架统一

Seer - Divided Rollout

  • 分段 Rollout 消除长 CoT 气泡
  • DGDS 动态 GPU 分配
  • Rollout/Training 交叉执行
  • 严格同步 On-Policy 保证

rl-swarm - P2P 去中心化

  • SAPO 去中心化 GRPO
  • 消费级 GPU 人人可参与
  • Hivemind P2P 梯度聚合
  • 无数据中心依赖

ThunderAgent - Program 感知

  • Program 抽象: 工作流级调度
  • KV-Cache 感知: 消除 Thrashing (7.1x)
  • 异步环境准备
  • 与 Slime/SkyRL 集成

DeepSeek V4 - OPD/FP4/WAL

  • OPD 替代混合 RL (10+ 教师蒸馏)
  • GRM: Actor 同时作为奖励模型
  • FP4 量化 (MXFP4→FP8 无损)
  • Token-WAL 容错 + DSec 沙箱
